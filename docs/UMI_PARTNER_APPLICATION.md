## Поля, которые заполняет владелец аккаунта

- логин;
- пароль и подтверждение пароля;
- ФИО по документам;
- номер телефона;
- согласие на обработку персональных данных;
- при запросе договора — паспортные, налоговые и платёжные реквизиты самозанятого.

Пароль нельзя сохранять в GitHub, переписке или публичных документах.

## Краткое описание специализации

Независимый разработчик прикладных модулей и автоматизации для CRM и интернет-магазинов. Основные направления: PHP/Node.js, API и webhooks, Telegram-интеграции, доставка, импорт и очистка CSV/XML/Excel, контроль заказов и бизнес-процессов, автоматические отчёты и компактные внутренние сервисы.

Работа выполняется письменно через GitHub. Закрытые доступы, токены и персональные данные не публикуются. Возможна оплата самозанятому НПД.

## Действия после отправки анкеты

1. Подтвердить email и сохранить данные аккаунта в менеджере паролей.
2. Письменно запросить назначение партнёрского менеджера.
3. Запросить NFR-лицензию Commerce.
4. Активировать выданный ключ в установленный UMI срок.
5. Уточнить письменный порядок договора и выплат самозанятому НПД.
6. Получить актуальный чек-лист UMI.Market.
7. Развернуть чистый NFR-контур без клиентских данных.
8. Проверить модуль по `prototypes/umi-order-telegram/SUBMISSION.md`.
9. Упаковать финальный релиз официальным экспортёром UMI.CMS.
10. Подать модуль на техническую и коммерческую модерацию.

## Что не делать

- не публиковать NFR-ключ, токен Telegram, chat ID и доступ к сайту;
- не загружать в GitHub резервную копию клиентского сайта;
- не использовать реальные персональные данные в тестах;
- не оплачивать необязательные услуги, пока менеджер письменно не подтвердит необходимость и условия;
- не соглашаться на звонок: все вопросы можно запросить письменно.
