# Test Plan — Orders API

## Цель

Подтвердить, что базовые операции создания и чтения заказа работают предсказуемо, некорректные запросы отклоняются понятными ответами, а ошибки можно диагностировать по идентификатору запроса.

## Объект

| Метод | Endpoint | Назначение |
|---|---|---|
| GET | `/health` | Проверка доступности |
| POST | `/api/v1/orders` | Создание заказа |
| GET | `/api/v1/orders/{id}` | Получение заказа |

## В области проверки

- HTTP-статусы, JSON и `Content-Type`;
- обязательность и формат `customer_email`;
- тип и диапазон `amount`;
- создание, идентификатор, статус и повторное чтение заказа;
- обработка неверного JSON, маршрута и идентификатора;
- наличие `X-Request-Id`;
- повторяемость тестов в чистом локальном окружении и CI.

## Вне области

- авторизация и роли;
- конкурентные запросы и нагрузка;
- постоянная база данных;
- платежи, персональные данные и внешние интеграции;
- безопасность production-инфраструктуры.

## Матрица проверок

| ID | Сценарий | Ожидаемый результат | Приоритет |
|---|---|---|---|
| API-001 | `GET /health` | 200, `status=ok`, JSON, request ID | Critical |
| API-002 | Валидное создание заказа | 201, уникальный ID, `created` | Critical |
| API-003 | Чтение созданного заказа | 200, данные совпадают | Critical |
| API-004 | Нет email | 422 с указанием поля | High |
| API-005 | `amount` = 0, отрицательный или boolean | 422 с указанием поля | High |
| API-006 | Неверный JSON | 400 `invalid_json` | High |
| API-007 | `text/plain` вместо JSON | 415 | Medium |
| API-008 | Неизвестный order ID | 404 `order_not_found` | High |
| API-009 | Неизвестный route | 404 `not_found` | Medium |

## Риски

- Несогласованность кодов 400 и 422 между командами.
- Случайное принятие boolean как числа в Python.
- Потеря диагностического request ID на ошибочных ответах.
- Зависимость тестов друг от друга через общее состояние.

## Критерии завершения

- Все critical/high проверки проходят.
- Тесты повторяемы локально и в GitHub Actions.
- Нет сетевых зависимостей и секретов.
- README содержит команду запуска и ограничения демонстрации.
