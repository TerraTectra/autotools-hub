# QA API Smoke Kit

Публичный учебный QA-кейс для откликов на Junior QA, Manual QA и Technical Support L2. Он показывает не коммерческий стаж, а проверяемый практический навык: анализ контракта API, позитивные и негативные проверки, воспроизводимые тесты, Postman и CI.

## Что проверяется

Локальный Orders API содержит три сценария:

- `GET /health` — состояние сервиса;
- `POST /api/v1/orders` — создание заказа;
- `GET /api/v1/orders/{id}` — чтение созданного заказа.

Автоматизированные проверки покрывают:

- статус-коды и JSON-контракт;
- обязательные поля и граничные значения;
- неверный JSON и неподдерживаемый `Content-Type`;
- неизвестные ресурсы и маршруты;
- сохранение и повторное чтение данных;
- диагностический заголовок `X-Request-Id`.

## Быстрый запуск

Требуется только Python 3.11+.

```bash
cd prototypes/qa-api-smoke-kit
python -m unittest -v test_api.py
```

Запуск API для ручной проверки:

```bash
python mock_api.py
```

После запуска API доступен на `http://127.0.0.1:8080`. Коллекцию `postman_collection.json` можно импортировать в Postman и выполнить вручную.

## Состав

- `mock_api.py` — детерминированный локальный API без внешних сервисов;
- `test_api.py` — smoke, contract и negative tests;
- `TEST_PLAN.md` — область тестирования, риски и критерии завершения;
- `BUG_REPORTS.md` — примеры оформления дефектов;
- `postman_collection.json` — ручные API-проверки с assertions;
- GitHub Actions — воспроизводимый запуск тестов на Python 3.11.

## Ограничения

Это демонстрационный проект. Примеры дефектов в `BUG_REPORTS.md` являются моделированием формата баг-репорта и не относятся к реальному продукту или заказчику. Никакой коммерческий опыт этим репозиторием не заявляется.
