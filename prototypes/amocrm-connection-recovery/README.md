# amoCRM Connection Recovery

Безопасный CLI для диагностики интеграции сайта с amoCRM после `401 Unauthorized`, истечения или отзыва токена.

## Возможности

- проверяет текущий Bearer token через `GET /api/v4/account`;
- объясняет коды `401`, `403`, `404`, `429` и временные ошибки;
- обновляет пару токенов через `POST /oauth2/access_token`;
- сохраняет новую пару в файл с правами `0600`;
- не выводит токены и секрет интеграции в журнал;
- не отправляет данные куда-либо, кроме указанного аккаунта amoCRM.

## Запуск проверки

```bash
MODE=check \
AMOCRM_BASE_URL=https://example.amocrm.ru \
AMOCRM_ACCESS_TOKEN=... \
node src/index.js
```

## Обновление токенов

```bash
MODE=refresh \
AMOCRM_BASE_URL=https://example.amocrm.ru \
AMOCRM_CLIENT_ID=... \
AMOCRM_CLIENT_SECRET=... \
AMOCRM_REFRESH_TOKEN=... \
AMOCRM_REDIRECT_URI=https://example.test/oauth \
node src/index.js
```

После успешного обмена старый refresh token больше нельзя использовать. Новую пару нужно сохранить атомарно, затем повторить проверку аккаунта.

## Границы

Инструмент не пытается обходить авторизацию. Если интеграция отключена, актуальный refresh token утрачен или уже использован, требуется повторное разрешение администратора аккаунта.

## Коммерческий формат

- диагностика и восстановление простой интеграции сайта — **5 000 ₽**, до одного рабочего дня;
- исправление хранения и автообновления токенов с тестовой заявкой — **8 000–12 000 ₽**;
- дальнейшая доработка обмена — после письменной оценки.
