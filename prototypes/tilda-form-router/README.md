# Tilda Form Router

Готовая техническая основа для приёма заявок из форм Tilda, мгновенного подтверждения webhook и дальнейшей доставки данных в Telegram и произвольную CRM.

## Что решает

- принимает стандартный POST от Tilda в формате `application/x-www-form-urlencoded`;
- поддерживает JSON для ручных тестов и внешних источников;
- отвечает `ok` сразу после сохранения заявки в локальную очередь;
- защищает endpoint секретом в URL или HTTP-заголовке;
- не отправляет cookie-данные в Telegram и CRM;
- нормализует имя, телефон, email, комментарий, форму, Lead ID и адрес страницы;
- не обрабатывает одну заявку повторно;
- сохраняет необработанные заявки между перезапусками;
- отправляет уведомление в Telegram;
- при необходимости передаёт унифицированный JSON во внешний CRM webhook;
- работает без внешних npm-зависимостей.

По документации Tilda webhook должен использовать HTTPS и ответить в течение пяти секунд. При отсутствии ответа Tilda выполняет ещё две попытки с интервалом в минуту. Этот прототип сначала сохраняет заявку и возвращает `ok`, а доставку выполняет из очереди.

## Запуск

```powershell
Copy-Item .env.example .env
# Заполните переменные окружения удобным способом.
$env:WEBHOOK_SECRET = "replace-me"
$env:DRY_RUN = "true"
npm test
npm start
```

Проверка:

```powershell
curl.exe -X POST "http://localhost:8080/webhooks/tilda?secret=replace-me" `
  -H "Content-Type: application/x-www-form-urlencoded" `
  --data-urlencode "tranid=demo-1" `
  --data-urlencode "formid=form123" `
  --data-urlencode "Name=Иван" `
  --data-urlencode "Phone=8 999 123-45-67" `
  --data-urlencode "Email=ivan@example.com" `
  --data-urlencode "Comments=Нужна интеграция с CRM"
```

Ответ:

```text
ok
```

## Переменные окружения

| Переменная | Назначение |
|---|---|
| `PORT` | Порт сервера, по умолчанию `8080` |
| `WEBHOOK_PATH` | Путь webhook, по умолчанию `/webhooks/tilda` |
| `WEBHOOK_SECRET` | Обязательный секрет |
| `WEBHOOK_SECRET_HEADER` | Заголовок секрета, по умолчанию `x-webhook-secret` |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота |
| `TELEGRAM_CHAT_ID` | Чат для уведомлений |
| `CRM_WEBHOOK_URL` | Необязательный endpoint CRM или промежуточного сервиса |
| `STATE_FILE` | Файл очереди и ключей дедупликации |
| `DRY_RUN` | `true` для вывода заявок в консоль без внешних отправок |

Должен быть настроен хотя бы один канал доставки: Telegram, `CRM_WEBHOOK_URL` или `DRY_RUN=true`.

## Подключение в Tilda

1. Развернуть сервис на HTTPS-адресе.
2. В настройках сайта открыть **Формы → Webhook**.
3. Указать адрес вида `https://example.ru/webhooks/tilda?secret=...`.
4. В нужном блоке формы включить приёмщик **WEBHOOK**.
5. Опубликовать страницу и отправить тестовую заявку.

Секрет рекомендуется делать длинным случайным значением. Не размещайте его в репозитории.

## Коммерческий формат

- Telegram-уведомления из одной формы — от 5 000 ₽;
- маршрутизация нескольких форм и унифицированный webhook для CRM — от 10 000 ₽;
- интеграция с конкретной CRM, дедупликацией, правилами распределения и журналом ошибок — от 15 000 ₽;
- white-label версия для студии — оценивается по составу поддерживаемых CRM и правилам установки.

Работа и согласования ведутся письменно. Для первичной оценки достаточно списка форм, примера обезличенных полей и описания нужного маршрута данных.
