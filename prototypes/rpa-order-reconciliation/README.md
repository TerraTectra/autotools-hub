# RPA Order Reconciliation Demo

Небольшой platform-neutral RPA-процесс на Python: робот сверяет заказы с успешными платежами, отделяет штатные результаты от бизнес-исключений и формирует файлы для следующего шага оркестрации.

Проект подготовлен как практический пример для вакансий и задач по RPA. Это не имитация опыта работы в конкретном дизайнере роботов: логика написана на Python и может быть вызвана из Sherpa RPA, cron, контейнера или другого оркестратора.

## Что показывает пример

- чтение двух CSV-источников;
- строгую проверку структуры и денежных значений;
- детерминированную сверку через `Decimal`;
- обнаружение отсутствующей оплаты;
- расхождение суммы;
- повторную успешную оплату;
- дубликаты заказов;
- оплату для неизвестного заказа;
- отдельную очередь бизнес-исключений;
- машиночитаемую JSON-сводку;
- unit-тесты без внешних зависимостей.

Все данные в `data/` синтетические.

## Запуск

Требуется Python 3.11+.

```bash
cd prototypes/rpa-order-reconciliation
python reconcile.py \
  --orders data/orders.csv \
  --payments data/payments.csv \
  --output output
```

Робот создаст:

- `output/reconciliation.csv` — результат по каждому уникальному заказу;
- `output/exceptions.csv` — очередь для ручной проверки или следующего робота;
- `output/summary.json` — сводка для оркестратора, мониторинга или уведомления.

Ожидаемая сводка демонстрационного набора:

```json
{
  "orders_processed": 4,
  "matched": 1,
  "business_exceptions": 4,
  "outcomes": {
    "AMOUNT_MISMATCH": 1,
    "DUPLICATE_PAYMENT": 1,
    "MATCHED": 1,
    "MISSING_PAYMENT": 1
  },
  "exception_codes": {
    "AMOUNT_MISMATCH": 1,
    "DUPLICATE_PAYMENT": 1,
    "MISSING_PAYMENT": 1,
    "ORPHAN_PAYMENT": 1
  }
}
```

## Тесты

```bash
python -m unittest discover -s tests -v
```

Тесты проверяют совпадение, отсутствие оплаты, расхождение суммы, дубли платежей, дубли заказов, неизвестные заказы и формирование выходных файлов.

## Как перенести в RPA-платформу

1. Оркестратор получает или выгружает входные CSV.
2. Python-шаг запускает `reconcile.py`.
3. При коде возврата `2` процесс фиксирует техническую ошибку входных данных.
4. При успешном запуске `summary.json` передаётся в мониторинг.
5. `exceptions.csv` направляется ответственному, в очередь задач или следующему роботу.
6. `reconciliation.csv` архивируется или загружается в целевую систему.

Бизнес-исключения не обрывают пакет: робот завершает обработку и отдаёт полный список проблем для управляемого ручного разбора.
