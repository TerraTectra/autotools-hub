# UIS Missed Call Recovery — proof of concept

Мини-сервис для возврата пропущенных входящих обращений из облачной телефонии UIS/совместимого callback-источника.

## Что делает

1. Принимает HTTP callback о звонке.
2. Нормализует разные варианты названий полей (`call_id`, `call_session_id`, `caller_number`, `call_status` и т. п.).
3. Определяет пропущенные входящие звонки.
4. Исключает повторную обработку одного события.
5. Отправляет ответственным уведомление в Telegram.
6. Опционально отправляет нормализованный JSON во внешний CRM webhook для создания лида или задачи.
7. Повторяет исходящие запросы при временной ошибке.

## Запуск

```bash
cp .env.example .env
set -a && . ./.env && set +a
npm start
```

Проверка:

```bash
npm run check
```

## Пример callback

```bash
curl -X POST 'http://localhost:8080/webhooks/uis' \
  -H 'content-type: application/json' \
  -H 'x-webhook-secret: replace-me' \
  -d '{
    "call_session_id": "demo-1",
    "call_status": "no_answer",
    "call_direction": "incoming",
    "caller_number": "+79991234567",
    "virtual_number": "+74951234567",
    "start_time": "2026-07-14T08:00:00Z"
  }'
```

## Коммерческий пилот

Пилот включает:

- адаптацию под точную схему callback конкретного аккаунта UIS;
- настройку защищённого endpoint;
- маршрутизацию Telegram по отделам или номерам;
- создание задачи/лида в CRM через webhook или API;
- журнал обработки, Docker-развёртывание и инструкцию;
- проверку на тестовых событиях.

Ориентир пилота: **15 000 ₽**, до пяти рабочих дней после предоставления тестовой схемы события и доступов.

## Ограничения PoC

Публичная часть не содержит реальных ключей, персональных данных и закрытой документации UIS. Точная карта полей callback и способ проверки подписи должны быть согласованы по документации, доступной в аккаунте клиента. До этого адаптер использует безопасный общий секрет и поддерживает распространённые алиасы полей.
