# Rental Listing Monitor Starter

Проверяемый каркас для проекта мониторинга новых объявлений аренды из нескольких источников с дедупликацией и отправкой структурированных карточек в Telegram.

Каркас подготовлен под публичный бриф Freelancehunt: OLX, LUN и Rieltor.ua. Он **не содержит обхода CAPTCHA, Cloudflare или иных защит** и не притворяется готовым парсером этих сайтов. Реальные адаптеры подключаются только после проверки доступных публичных страниц/API, правил источников и согласования критериев с заказчиком.

## Что уже реализовано

- единая модель объявления для разных сайтов;
- стабильный ключ объявления и SQLite-дедупликация;
- фильтры по цене, городу, району, комнатам и ключевым словам;
- структурированная Telegram-карточка;
- журнал новых и уже отправленных объявлений;
- dry-run с демонстрационными источниками;
- безопасная отправка через Telegram Bot API;
- тесты нормализации, дедупликации и фильтров.

## Быстрый запуск

```powershell
cd prototypes/rental-listing-monitor
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python rental_monitor.py --demo --dry-run
pytest -q
```

## Telegram

Секреты передаются только через переменные окружения:

```powershell
$env:TELEGRAM_BOT_TOKEN = "..."
$env:TELEGRAM_CHAT_ID = "..."
python rental_monitor.py --demo
```

## Подключение реальных источников

Для каждого сайта нужен отдельный адаптер, возвращающий `Listing`. Перед реализацией необходимо письменно определить:

1. Ссылки на поиски или правила фильтрации.
2. Частоту проверки.
3. Поля карточки: цена, адрес, фото, описание, автор, телефон, ссылка, время публикации.
4. Разрешённый способ получения данных: официальный API, открытая лента, публичные страницы или браузерный сценарий без обхода проверок.
5. Поведение при CAPTCHA/блокировке: остановка и уведомление, а не обход.

## Границы безопасности

- не обходить CAPTCHA, Cloudflare и лимиты источников;
- не собирать лишние персональные данные;
- не хранить токены и cookies в Git;
- не отправлять одно объявление повторно;
- при изменении разметки или блокировке останавливать источник и сообщать об ошибке.
