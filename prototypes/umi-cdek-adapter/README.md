# UMI.CMS → CDEK Adapter Scaffold

Безопасная заготовка модуля доставки СДЭК для UMI.CMS, созданная под запрос действующего UMI-интегратора.

## Готово

- payload расчёта одного тарифа и списка тарифов;
- фильтры ПВЗ по официальным полям SDK;
- payload создания заказа;
- проверка обязательных данных получателя, адресов, упаковок и товаров;
- нормализация российских телефонов;
- идемпотентность по номеру и ревизии заказа;
- атомарное локальное состояние с правами `0600`;
- перечень live-операций официального SDK;
- тесты без внешних API и клиентских секретов.

## Не реализовано без тестового контура

- адаптер конкретной версии и структуры UMI.CMS;
- административная форма настроек;
- выбор ПВЗ в оформлении заказа;
- реальные запросы в API СДЭК;
- webhook статусов;
- печатные формы;
- правила наложенного платежа, НДС и упаковки конкретного магазина.

Административная форма с доступами API, городом отправления и рабочими параметрами входит во все production-варианты. Она не является отдельной функцией только дорогой универсальной версии.

## Проверка

```bash
php tests/run.php
```

## Live-транспорт

Для реальных запросов устанавливается официальный PHP SDK:

```bash
composer require cdek-it/cdek-sdk2.0
```

Доступы СДЭК и данные магазина не размещаются в репозитории. Перед production-разработкой нужны тестовый сайт UMI.CMS, тестовый кабинет СДЭК, версия PHP, структура заказа и согласованные сценарии тарифа, ПВЗ и курьерской доставки.

## Коммерческий статус

Это техническая заготовка, а не готовый универсальный модуль. Для стандартного магазина UMI.CMS без полностью переписанного checkout предлагаются фиксированные варианты:

- **20 000 ₽, 5–7 рабочих дней** — расчёт стоимости и срока, доставка до двери и ПВЗ списком, административные настройки API/города/тарифов, сохранение доставки в заказе и журнал ошибок;
- **35 000 ₽, 8–12 рабочих дней** — всё выше плюс карта ПВЗ, создание и отмена отправления, получение статуса;
- **55 000 ₽, 12–18 рабочих дней** — повторно устанавливаемая white-label версия для клиентских проектов интегратора, с установщиком, административными настройками и документацией.

Отдельный платный discovery для стандартной задачи не требуется. До согласования сделки можно бесплатно письменно проверить структуру checkout и сообщить, влияет ли нестандартная реализация на цену. Оплата: 50% до начала production-адаптации, остаток после демонстрации на тестовом контуре.
