# UMI.Market submission checklist

Этот репозиторий содержит технический proof of concept и review-архив, но не официальный пакет UMI.Market.

## Уже выполнено

- уникальное имя компонента `terratectra_order_telegram`;
- штатный listener `order-status-changed`;
- конфигурация через реестр;
- персональные данные отключены по умолчанию;
- изоляция ошибок и повтор временно неудачной отправки;
- проверка PHP-синтаксиса;
- шесть автоматических тестов;
- поиск случайно опубликованных секретов;
- воспроизводимый review-архив с SHA-256.

## Требуется NFR-контур UMI.CMS

- установить компонент через `/admin/config/modules`;
- проверить параметры события и ссылку на объект заказа на поддерживаемой версии UMI.CMS;
- проверить значения реестра после установки;
- отправить тестовое уведомление по синтетическому заказу;
- проверить совместимость с нужными редакциями Shop, Commerce и Ultimate;
- добавить штатную административную форму настроек согласно целевой версии;
- проверить обновление и удаление компонента;
- протестировать PHP-версии, официально поддерживаемые целевой версией UMI.CMS.

## Требуется партнёрский доступ

- подтвердить формат выплат самозанятому партнёру;
- получить NFR-лицензию у назначенного менеджера;
- получить актуальный чек-лист модерации;
- упаковать релиз официальным экспортёром UMI.Market;
- подготовить скриншоты, условия поддержки, совместимость и коммерческую цену;
- отправить продукт через партнёрский кабинет.

## Безопасность

Нельзя помещать в GitHub токен бота, chat ID, клиентские данные, production-домен или архив сайта. Для тестов используются отдельный бот и синтетические заказы.
