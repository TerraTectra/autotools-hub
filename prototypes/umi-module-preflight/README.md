# UMI.CMS Module Preflight

Локальная статическая проверка пакета компонента UMI.CMS перед NFR-тестом и подачей в UMI.Market.

Это **не официальный валидатор UMI.CMS** и не заменяет установку на поддерживаемую версию системы. Инструмент ловит распространённые ошибки упаковки до передачи пакета клиенту или модератору.

## Что проверяет

- наличие и разбор `install.php`;
- имя компонента, `config` и `default_method_admin`;
- существование файлов из `$COMPONENTS`;
- наличие типовых файлов компонента;
- регистрацию `umiEventListener`;
- PHP-синтаксис, если доступен PHP CLI;
- случайно включённые `.env`, SQL-дампы, приватные ключи и значения, похожие на секреты;
- UTF-8 BOM в текстовых файлах;
- архивы и резервные файлы внутри пакета;
- наличие README с инструкцией.

## Запуск

```bash
python preflight.py /path/to/module-package
```

JSON-отчёт:

```bash
python preflight.py /path/to/module-package --json report.json
```

Строгий режим, где предупреждения дают ненулевой код завершения:

```bash
python preflight.py /path/to/module-package --strict
```

Коды завершения:

- `0` — ошибок нет;
- `1` — только предупреждения в строгом режиме;
- `2` — найдены ошибки.

## Проверка проекта

```bash
python -m unittest discover -s tests -v
python preflight.py ../umi-order-telegram/module --json /tmp/umi-preflight.json
```

## Коммерческий формат

- автоматический отчёт по одному пакету — **2 500 ₽**;
- отчёт и исправление типовых ошибок упаковки — **5 000 ₽**;
- адаптация проверки под внутренние правила веб-студии — от **10 000 ₽**.

Проверка выполняется локально. Рабочие доступы к сайту для статического аудита не требуются.
