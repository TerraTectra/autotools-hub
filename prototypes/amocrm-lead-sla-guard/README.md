# amoCRM Lead SLA Guard

Proof of concept для контроля скорости обработки сделок amoCRM.

Сервис регулярно получает сделки через API v4, находит сделки без будущей задачи, которые не изменялись дольше настроенного SLA, и уведомляет ответственного в Telegram. Опционально создаёт задачу «Звонок» в карточке сделки.

## Что реализовано

- получение сделок через `GET /api/v4/leads`;
- постраничная загрузка с ограничением по окну актуальности;
- SLA по связке `pipeline_id:status_id`, по статусу или общий порог;
- исключение закрытых статусов;
- проверка `updated_at` и `closest_task_at`;
- Telegram-уведомление со ссылкой на карточку;
- опциональное создание задачи через `POST /api/v4/tasks`;
- дедупликация до следующего изменения сделки;
- повторные попытки при `429` и `5xx`;
- Docker, unit-тесты и GitHub Actions.

## Запуск

```bash
cp .env.example .env
set -a && source .env && set +a
npm start
```

Проверка:

```bash
npm run check
```

## Настройка SLA

```json
{
  "default": 1800,
  "123456:654321": 900,
  "654322": 3600
}
```

Ключ `pipeline_id:status_id` имеет максимальный приоритет, затем применяется правило по `status_id`, затем `default`.

## Коммерческий пилот

Пилот может включать:

- OAuth 2.0 и автоматическое обновление токенов;
- настройку реальных воронок и этапов;
- разные правила по источнику, ответственному и времени суток;
- маршрутизацию уведомлений по командам;
- автоматические задачи и эскалации;
- PostgreSQL/Redis вместо локального JSON;
- развёртывание, журналирование и инструкцию администратора.

Ориентир пилота: от 15 000 рублей, до 5 рабочих дней после получения тестового аккаунта и согласования критериев приёмки.

- [Бриф коммерческого пилота без production-доступа](CLIENT_BRIEF_RU.md)
- [Готовое обращение в поддержку amoCRM и отдел интеграций](TECH_SUPPORT_REQUEST_RU.md)

Публичный PoC не содержит токенов или персональных данных. Для production необходимы тестовая интеграция amoCRM, OAuth-настройки и согласованная политика обработки данных.
