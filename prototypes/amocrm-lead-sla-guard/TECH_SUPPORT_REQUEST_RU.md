# Обращение в поддержку amoCRM по интеграции

Официальная документация amoCRM указывает, что запрос технического аккаунта и связь с отделом интеграций выполняются через чат поддержки внутри зарегистрированного аккаунта.

Этот документ не содержит паролей, ключей доступа или данных клиентов.

## 1. Короткое первое сообщение в чат

```text
Здравствуйте. Нужен чат с отделом интеграций.

Разрабатываю внешний сервис amoCRM Lead SLA Guard: он через API v4 контролирует сделки без актуальной задачи или без движения дольше заданного SLA, отправляет уведомление ответственному и при необходимости создаёт задачу в карточке сделки.

Нужно определить подходящий формат публикации и сотрудничества: внешняя интеграция либо публичная интеграция с техническим аккаунтом. Также прошу письменно уточнить, допускается ли разработчик в статусе самозанятого НПД без ИП и юридического лица.

Коммуникацию прошу вести в чате или по электронной почте, без звонка.
```

## 2. Расширенное описание для отдела интеграций

```text
Здравствуйте.

Разрабатываю amoCRM Lead SLA Guard — сервис контроля скорости обработки сделок.

Что уже реализовано:
— получение сделок через API v4;
— SLA по воронкам и этапам;
— обнаружение сделок без будущей задачи и без изменений дольше установленного срока;
— исключение закрытых этапов;
— Telegram-уведомления со ссылкой на карточку;
— опциональное создание задачи ответственному;
— защита от повторных уведомлений;
— повтор запросов при временных ошибках;
— Docker, автоматические тесты и GitHub Actions.

Публичные материалы:
— продукт: https://github.com/TerraTectra/autotools-hub/tree/main/prototypes/amocrm-lead-sla-guard
— страница пилота: https://terratectra.github.io/autotools-hub/site/amocrm.html
— бриф пилота: https://github.com/TerraTectra/autotools-hub/blob/main/prototypes/amocrm-lead-sla-guard/CLIENT_BRIEF_RU.md

Прошу помочь определить правильный формат:
1. Можно ли развивать это решение как внешнюю интеграцию без JS-виджета?
2. Возможна ли публикация или представленность такого решения в экосистеме amoCRM?
3. Если необходима публичная интеграция, можно ли получить технический аккаунт и чат с отделом интеграций?
4. Допускается ли автор и получатель выплат в статусе физического лица — самозанятого НПД, без ИП и юридического лица?
5. Какие документы, демонстрационные материалы и требования к поддержке нужно подготовить?
6. Можно ли пройти рассмотрение и согласование полностью письменно в чате или по электронной почте?

Production-доступы и данные клиентов в публичный репозиторий не помещаются. Для проверки готов использовать отдельный тестовый аккаунт и синтетические сделки.

Контакт:
Telegram: @tahioff
Email: nikidom123@gmail.com
```

## 3. Что приложить только по запросу поддержки

- ссылку на конкретный зелёный запуск GitHub Actions;
- перечень API-методов и необходимых прав;
- схему OAuth-подключения;
- короткое видео или набор скриншотов тестового сценария;
- условия пилота и поддержки;
- подтверждение статуса самозанятого без публикации ИНН в открытом репозитории.

## 4. Что не отправлять в чат до назначения специалиста

- доступ к рабочей CRM;
- данные реальных клиентов и сделок;
- приватные OAuth-данные;
- доступ к серверу;
- данные Telegram-бота;
- документы, не относящиеся к проверке статуса.
